The Router

The Router is a lookup table. It maps a URL pattern and HTTP method to a handler function. When a request arrives, Axum finds the first matching row and calls that function. Everything else gets a 404 automatically.

Route table

The highlighted row shows GET /movies/42 matching GET /movies/{id} - the {id} wildcard captures 42.

Tip

The {id} wildcard — any segment in curly braces {} is a named wildcard. Axum captures whatever value is in that position and makes it available to your handler via the Path extractor. You will see this in section 4.
