Request journey

A request enters from the bottom — Tokio receives raw bytes, Hyper parses them into HTTP, Tower runs middleware, Axum routes to your handler. The response travels back the same way.

✍️

What you actually write

You only interact with Axum and a few Tower helpers. You never touch Hyper or Tokio directly except for the two lines that start the server.
