Handlers

A handler is an async fn the router calls when a route matches. It receives typed data through extractors and returns anything that implements IntoResponse. Axum figures out the rest.

1
Must be async fn
All Axum handlers are async — they run on the Tokio runtime
2
Arguments must be extractors
Every argument implements FromRequest — Axum calls them automatically
3
Return type: IntoResponse
Axum calls into_response() on whatever you return

Simplest handler

No inputs. Returns a string. Axum produces a 200 OK with text/plain body.
